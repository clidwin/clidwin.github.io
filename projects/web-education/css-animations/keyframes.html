CSS Animations: Keyframes

Keyframes dictate when changes occur and what changes are being made in an animation. If you have used animation applications such as Autodesk Maya, Adobe After Effects, or Maxon's Cinema 4D before, keyframes have the same functionality in these programs as they do in CSS3 animations.

This section will go over the syntax for keyframes in CSS3 as well an introduction to what content could be animated with keyframes. Just like with animation applications, though, creating keyframes does not automatically create animations. The next section will discuss how to activate animations, or in other words explain what is used instead of a "play button" in an animation application to show the keyframes in motion.

Vendor Prefixes

The keyframes selector is, as of this module's writing, an experimental feature, and therefore requires Vendor Prefixes. These prefixes go in between the @ symbol and "keyframes", so to use keyframes for Firefox, the syntax would be @-moz-keyframes.

Keyframe Syntax

Keyframes are set up syntactically by using the @keyframes selector followed by a name you create to represent the keyframes (This is similar to the @media selector used for responsiveness, as opposed to properties like background-color). The variable name you create can be any alphanumeric string (with underscores) as long as it starts with a letter and is not already used by another selector (in other words, you cannot name an animation "media"), but by convention the naming is uppercase and has a name that relates to the action being animated. An example keyframe name for an animation about a bouncing ball might be BOUNCE.

Just like with other selectors, after the variable name are open and closing curly braces ({}), which encapsulate the CSS rules for the keyframe.

Timing Rules

As discussed in the last topic, the @keyframes [INSERT_NAME] {} syntax encapsulates a set of CSS rules, each of which are a keyframe (or set of keyframes) that contain the properites you would like to have applied at that point in the animation. Specifically, these CSS rules relate to the time of the animation, and can come in two forms: percentages in the inclusive range of 0% and 100%, and the selectors from{} and to{}.

Percentages are flexible and are called by listing the value of the percent, followed by the percentage, and concluded with the opening and closing curly braces. Any percentage value can be chosen, and the same properties can be applied to multiple percentages by grouping the percentages with commas such as 0%, 100%, just as you can write CSS rules that apply to multiple selectors.

A simpler but less flexible option to the percentage timing rules is from{} and to{}. from{} contains properties that are applied at the start of the animation, and to{} contains properties that are applied at the end of the animation. This option is useful if there are only two states for an animation, but these rules cannot be mixed with percentage timing rules.

In both cases of timing rules, CSS calculates intermediate steps of the animations to show a smooth transition between property values. Similar to animation programs, this means we only need to define the points where we want an effect applied, and CSS3 will calculate the transitions for us.

Keyframed Content

Just like with regular CSS selectors, properties such as background-color, margin, and width can all be specified inside a timing rule.
