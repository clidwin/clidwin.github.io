CSS Animations: Keyframes

Keyframes dictate when changes occur and what changes are being made in an animation. If you have used animation applications such as Autodesk Maya, Adobe After Effects, or Maxon's Cinema 4D before, keyframes have the same functionality in these programs as they do in CSS3.

This section will go over the syntax for keyframes in CSS3 as well an introduction to what content could be animated with keyframes. Just like with animation applications, though, keyframes do not automatically play animations. The next section will discuss how to activate animations, or in other words explain what is used instead of a "play button" to show the keyframes in motion.

@keyframes [INSERT_NAME] {}

Keyframes are set up syntactically by using the @keyframes[1] selector followed by a name you select to represent the keyframes (This is similar to the @media selector used for responsiveness, as opposed to properties like background-color). The variable name you get to select can be any alphanumeric string as long as it starts with a letter and is not already used by another selector, but by convention the naming is uppercase and has a name that relates to the action being animated.

After the variable name are open and closing curly braces ({}), which open and close the declaration part of the keyframe CSS rule. All other content relating to keyframes will be contained inside these braces.

Timing Rules

The @keyframes [INSERT_NAME] {} syntax encapsulates a set of CSS rules, each of which are a keyframe (or set of keyframes) that contain the properites you would like to have applied at that point in the animation. Specifically, these CSS rules relate to the time of the animation, and can come in two forms: percentages in the inclusive range of 0% and 100%, and the selectors from{} and to{}.

Percentages are flexible and are called by listing the value of the percent, followed by the percentage, and concluded with the opening and closing curly braces. Any percentage value can be chosen, and the same properties can be applied to multiple percentages by grouping the percentages with commas such as 0%, 100%.

A simpler but less flexible version of timing rules is from{} and to{}. from{} contains properties that are applied at the start of the animation, and to{} contains properties that are applied at the end of the animation.

In both cases of timing rules, CSS calculates intermediate steps of the animations to show a smooth transition between property values.

Keyframed Content

Just like with regular CSS selectors, properties such as background-color, margin, and width can all be specified inside a timing rule.

Footnotes

[1] The keyframes selector is, as of this module's writing, an experimental feature, and therefore requires Vendor Prefixes. These prefixes go in between the @ symbol and "keyframes", so to use keyframes for Firefox, the syntax would be @-moz-keyframes.
