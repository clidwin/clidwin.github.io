CSS Animations: Prefixes for Animations

Before new web features are released, they are often developed in or experimented with different browsers. During this "beta" phase, the features can be enabled using special tags called vendor prefixes, which are prepended to the feature to tell the browser to treat the feature as an experiment.

While CSS animation is becoming more widely used, it is unfortunately not a fully released feature, which means in most cases vendor prefixes are required for both development and viewing CSS animations.

Vendor Prefix Syntax

Below is a list of the most common vendor prefixes and their affiliated browsers:

  • -webkit- is used by Chrome and Safari.

  • -moz- is used by Firefox.

  • -o- is used by Opera.

As you might anticipate from the "prefix" naming, these prefixes are added just before the feature you would like to use. So, for example, in doing an animation, the syntax for using animation in a Chrome browser would be -webkit-animation.

Experimental Features

More information on why we use prefixes and prefixes for other browsers can be found at OpenWeb's Vendor Prefixes webpage.

Note for the Following Sections

In all other sections for this module, vendor prefixes need to be applied. Highlighted syntax and terminology will be displayed without these prefixes, though, so that the documentation supports CSS animations when they become a fully-supported feature.
