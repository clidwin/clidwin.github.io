CSS Animations: Animation Subproperties

After keyframes are created, the next step is to call the keyframes and give parameters to the animation so that it can become an active element in the webpage. These parameters include timing, repetition, and directional elements, are set for the animation through animation sub-properties.

To apply animations to elements, pick the element(s) the animation will be applied to, and at minimum include the animation-name, animation-duration, and animation-iteration-count sub-properties along with the other CSS styling you would like to apply. Also note that for all of the subproperties, vendor prefixes are needed.

animation-name

The subproperty that links an animation to its keyframe selector is animation-name. Its value is the name given alongside the keyframes selector.

As an example, if a keyframe selector was set up with the variable name "bounce", the associated call to the animation would be animation-name: bounce;.

animation-duration

animation-duration describes how long the animation lasts, or in other words the amount of time between 0% and 100% or the from and to keyframes. Its value is a positive integer indicating the amount of time, followed by the character 's' to denote that the time is in seconds or 'ms' to denote the time is in milliseconds.

As an example, if an animation should last a total of 5 seconds, the subproperty text would be animation-duration: 5s;.

animation-iteration-count

To specify the number of times an animation should be executed, use theanimation-iteration-count subproperty. There are two possible values for this sub-property: the keyword 'infinite', which means play the animation as long as the page is opened on a browser, or a positive integer, which indicates the animation will occur that integer's number of times before returning to its starting state.

As an example, if an animation should occur 6 times before stopping, the subproperty text would be animation-iteration-count: 6;.

animation-direction

By default, CSS3 animations are played from 0% to 100% (or from to to), and then directly return to 0% (or from). Because this sometimes creates abrupt animation, the animation subproperty animation-iteration-count provides a way to play the animation forward (from 0% to 100%) and then in reverse (100% to 0%).

The direction subproperty has two values: 'normal' to indicate that the animation should always be played from 0% to 100%, or 'alternate' to indicate the animation should switch between playing the animation in incrementing or decrementing percentage order. As an example, animation-direction: alternate;.

animation-delay

To put a pause before the start of an animation, use the animation-delay subproperty. Just like duration, the value needs to be a positive integer followed by the specification 's' for second and 'ms' for millisecond.

As an example, if an animation should start after 3 seconds, this could be done through animation-delay: 3s;.
