CSS Transforms: Origins

By default, transformations are done in the center of an element (50% width, 50% height, and 0% depth). However, we are not restricted in CSS to doing transformations on the center of objects. We can place the transformation anywhere within an element using the property transform-origin.

transform-origin

Syntactically, the transform-origin property can take 1, 2, or 3 values. In the one value model, the length provided describes the horizontal and vertical positioning of the origin, and the offsets are based on the top left corner of the element. In the two value model, the first value is the horizontal offset for the origin, and the second argument is the vertical offset for the origin. The three value model uses x, y, and z as the offset for the origin.

Just like the margin property, it is possible to combine length values and keywords such as top, bottom, left, right, and center. When using a keyword, the keyword does not need to go in the order specified above; it can appear out of order (except in the case of center), but as a good rule of thumb, follow the convention of listing the x offset first, followed by the y offset, followed by the z offset as applicable.

As an example, if someone wanted to rotate a square from its bottom right corner, they might use the CSS transform-origin: right bottom;.
