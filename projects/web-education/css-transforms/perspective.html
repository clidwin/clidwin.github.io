CSS Transforms: Perspectives

By default, DOM elements are drawn on an xy-plane, which creates a flat panel on which objects are created and rendered. A third dimension does exist on web pages, but it is set to have a value of zero by default, thus creating the illusion of a two dimensional surface.

perspective allows us to set the space for this third dimension, and exposes a new set of transform functions and capabilities.

perspective

We use the perspective tag to set a depth or distance a focal point is perceived to be from the page. It is a similar concept to a camera in a 3D modelling program because all child elements are created based on the perspective's origin. To reiterate, this tag is applied to the parent element of the element being transformed, rather than the transformed element itself.

As an example, if we set a perspctive to 200px and had child elements all rotated in the y-direction by 30 degrees, the perspective might view the rotated element differently than its originally rendering based on the perspective's view of the element. We could create this perspective through the CSS perspective: 200px;.

By default, the perspective's origin, or point of reference, is set to be the center of the element. We can change this using the perspective-origin property, which functions the same way transform's origin property does.

Transforms in 3D

In the case of the matrix, rotate, scale, and translate functions (in other words, everything but skew), it's possible to add the characters "3d" to the end of these function and have a new transform function that adds a third dimension effect. For scale and translate, these functions now take 3 arguments instead of two: an x value, a y value, and a z value. An example might be translating by 20px in all three dimensions, which would be the CSS transform: translate3d(20px, 20px, 20px);.

To rotate in 3d rotate, we use four arguments to the new function. The first three arguments are for x-position, y-position, and z-position for the rotation point, and the forth argument is the angle of rotation for the shape. As an example, if we wanted to apply a 30 degree rotation to a point 5 pixels in the x, y, and z directions, we could do so through the CSS transform: rotate3d(5px, 5px, 5px, 30deg);

In the context of matricies, the 3d version of the function expands the arguments to outline a four by four matrix, and the arguments are ordered by column. The first three columns (first twelve arguments) are all numbers describing the transformations, and the last column (last four arguments) describe the translation effect. For more information, visit the Mozilla Developer Network entry on the topic.

transform-style

The transform-style property only has three possible values. preserve-3d emphasizes that elements should be rendered in 3d space, whereas flat overrides 3d properties and renders elements as if they all existed on the same xy plane. inherit is a keyword we have seen before in other properties, and says to use the style applied in the closest parent element.

CSS-Tricks has an excellent example showcasing how this property affects element rendering.
